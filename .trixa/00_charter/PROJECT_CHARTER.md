# Project Charter

## TL;DR
- Viva Engage風の社内SNSプラットフォーム「Ummy」を構築
- 技術的な議論を1箇所に集約し、非同期コラボレーションを促進
- コミュニティ機能と投稿機能（コメント・リアクション）を核とする
- 現状: Next.js + Figmaベースで見た目実装完了（技術的負債あり）、データ層は未実装
- 既存実装のリファクタリング + 新機能開発を並行

## Status
Accepted

## Problem
- 技術的な議論が複数ツール（Slack、メール、Wiki、ドキュメント）に分散している
- 過去の議論・知見が埋もれやすく、検索・参照が困難
- リアルタイムチャットでは非同期コラボレーションに限界がある
- コミュニティ単位での情報整理が不十分

## Goal
- 社内技術者が技術的な議論・知識共有の第一選択肢として Ummy を利用する
- 重要な技術議論がコミュニティ単位で蓄積され、検索可能な状態を実現
- 非同期でも建設的な議論ができる環境を提供
- （測定可能な目標）本格運用開始3ヶ月後、技術者の週次アクティブ率 70% 以上

## Scope
- In:
  - **既存実装のリファクタリング**（UI/ロジック分離、ベストプラクティス準拠）
  - コミュニティ機能（作成、参加、一覧表示、コミュニティ別フィード）
  - 投稿機能（テキスト投稿、添付ファイル対応は将来）
  - コメント機能（ネストした返信を含む）
  - リアクション機能（いいね、祝福、ひらめき、笑など複数種類）
  - ユーザープロフィール（名前、役職、アバター）
  - 検索機能（投稿・コミュニティ検索）※優先度高
  - 通知機能（コメント・リアクション通知）
  - ピン留め機能（重要投稿の固定）
  - Webブラウザ対応（デスクトップ優先）
  - AWS本番環境デプロイ対応（AWS CLI使用）
  - ローカル開発環境構築
- Out:
  - ダイレクトメッセージ（DM）機能
  - ビデオ会議・音声通話機能
  - 外部SNS連携（Twitter、LinkedIn等）
  - モバイルアプリ（初期フェーズ）
  - 詳細な権限管理（初期は参加/非参加のみ）
  - コンテンツのバージョン管理・編集履歴（初期フェーズ）
  - マルチクラウド対応（初期は AWS 単一）

## Stakeholders
- **主要利用者**：社内技術者（エンジニア、デザイナー、プロダクトマネージャー）
- **副次利用者**：技術議論に関心のある他部門メンバー
- **管理者**：ITチーム、プラットフォーム運用チーム
- **開発チーム**：本プロジェクト開発メンバー

## Constraints (high level)
- **技術制約**：
  - Next.js 16 (App Router)、React 19、TypeScript 5 を採用
  - Tailwind CSS 4、Motion (Framer Motion後継) を採用
  - 現状はフロントエンドのみ実装済み（見た目のみ）
  - データ永続化層（DB、API）は未実装
  - **技術的負債**：既存実装はベストプラクティス未準拠（UI/ロジック未分離、600行超コンポーネント、next/image未使用等）。段階的にリファクタリング予定
- **インフラ制約**：
  - 開発環境：ローカル実行を前提
  - 本番環境：AWS 上で運用
  - デプロイ方式：AWS CLI を使用（IaC、CI/CD の詳細は別途定義）
  - クラウドベンダーロックイン：AWS に依存（初期フェーズ）
- **スコープ制約**：
  - 初期フェーズは最小機能（MVP）に絞る
  - モバイルアプリは対象外（Webレスポンシブのみ）
- **運用制約**：
  - 社内専用（認証必須）
  - セキュリティ・プライバシー要件は社内規定に準拠
  - 本番環境は AWS 上で可用性・スケーラビリティを確保
- **リソース制約**：
  - 開発リソースは限定的（詳細は別途定義）
  - 早期フィードバック重視（段階的リリース）
