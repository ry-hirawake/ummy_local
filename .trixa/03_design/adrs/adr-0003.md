# ADR-0003: 永続化基盤は Aurora PostgreSQL Serverless v2 と Route Handlers + Service Layer を採用する

## TL;DR
- Decision: 永続化基盤は Amazon Aurora PostgreSQL Serverless v2 を採用する
- API Boundary: Next.js App Router の Route Handler を HTTP 境界とし、その下に Service Layer を置く
- Secrets: AWS 本番の秘密情報は Secrets Manager で扱い、UI から DB へ直接触れない
- Why: Ummy のドメインは関係性が強く、AWS 上での可変負荷と運用性を両立しやすい

## Status
Proposed

## Decision
- 永続化基盤は Amazon Aurora PostgreSQL Serverless v2 を採用する
- コアエンティティは `User`, `Community`, `Membership`, `Post`, `Comment`, `Reaction`, `Notification` を関係モデルで保持する
- Next.js App Router の `app/**/route.ts` を HTTP 境界として使用し、DB アクセスは Route Handler 直下の Service Layer へ集約する
- React Component から DB へ直接アクセスしない
- AWS 本番の DB 接続情報やアプリ秘密情報は Secrets Manager で管理し、ローカル開発との差分はサーバー側設定で吸収する

## Context
- Ummy はコミュニティ所属、参加関係、コメントスレッド、リアクション、通知など相互参照の多いドメインを持つ
- Requirements の `FR-10` と Constraints の `TC-4` は永続化/API 設計を要求している
- ユーザー数と初期フェーズの開発リソースを考えると、モデリング容易性と運用自動化の両立が重要である
- Next.js 公式の Route Handler は App Router 内でカスタム request handler を作る標準手段であり、UI と API を同一リポジトリ内で整理しやすい

## Options Considered
### Option A: Aurora PostgreSQL Serverless v2 + Route Handlers + Service Layer
- Pros:
  - Membership, Comment, Notification のような関係データを素直に表現できる
  - Aurora Serverless v2 は on-demand かつ autoscaling で、変動負荷や開発/検証環境に向く
  - Route Handler は Next.js App Router の標準 API 境界として扱える
  - Service Layer にビジネスルールを閉じ込めやすい
- Cons:
  - SQL スキーマ設計と migration 管理が必要
  - Aurora 運用コストを DynamoDB より深く理解する必要がある

### Option B: DynamoDB + Route Handlers
- Pros:
  - フルマネージドで高スケールに向く
  - 単純な key-value / document access では強い
- Cons:
  - 多対多の Membership、コメントスレッド、通知集約、検索前処理などで access pattern の先読みが重い
  - 初期段階で single-table 設計まで固定すると仕様変更耐性が下がる

### Option C: 別 BFF / 別 API サービスを先に分離する
- Pros:
  - 将来の多クライアント化には有利
  - Next.js から責務を分けやすい
- Cons:
  - MVP 段階でデプロイ対象と運用面を増やす
  - 同一チーム・単一 Web クライアント前提では過剰分割になりやすい

## Rationale
- Ummy の主要ユースケースは relational join と整合性制約が中心であり、Aurora PostgreSQL の方がモデルとクエリの自然さが高い
- AWS 公式ドキュメント上、Aurora Serverless v2 は on-demand / autoscaling で variable workload や development and test systems に向く
- RDS for PostgreSQL は managed backups や VPC 内配置などの運用基盤が揃っており、AWS 本番前提と整合する
- Next.js の Route Handler は App Router 内で custom request handler を作る標準手段であり、別 BFF を増やさず責務を明示できる
- Secrets Manager により hard-coded secret を避け、ランタイムで安全に秘密情報を扱える
