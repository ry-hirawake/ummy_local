# ADR-0004: Next.js 本番配置は ECS Fargate + ALB + CloudWatch Logs を採用する

## TL;DR
- Decision: Ummy の Next.js 本番実行基盤は ECS Fargate とする
- Ingress: HTTPS は Application Load Balancer で受け、Fargate task へ流す
- Operations: コンテナログは CloudWatch Logs、秘密情報は Secrets Manager を正とする
- Why: SSR/RSC を保った self-hosting と AWS 標準運用を両立しやすい

## Status
Proposed

## Decision
- Ummy の Next.js アプリケーションはコンテナ化して ECS Fargate 上で稼働させる
- 外部公開入口は HTTPS 終端を持つ Application Load Balancer とする
- Fargate service は ALB 配下で health check と autoscaling を前提に運用する
- コンテナログは `awslogs` driver で CloudWatch Logs へ集約する
- 本番秘密情報は Secrets Manager からコンテナへ注入する
- デプロイ手順は IaC で基盤を管理しつつ、実行は AWS CLI ベースの標準手順に統一する

## Context
- Requirements の `NFR-2`, `NFR-7` と Constraints の `TC-2`, `OC-4` は AWS 本番運用、監視、ログ、ロールバックを要求している
- Next.js は `next start` による self-hosting で middleware や image optimization を含む実行形を維持できる
- Ummy は認証付き SSR/RSC アプリであり、静的ホスティングだけでは足りない
- 初期フェーズでも health check、ログ集約、秘密情報分離を標準化しておく必要がある

## Options Considered
### Option A: ECS Fargate + ALB + CloudWatch Logs
- Pros:
  - 長時間稼働する stateless web service として運用しやすい
  - ECS service は task 障害時の再スケジュールを担い、ALB と組み合わせて health check を取れる
  - ALB は HTTP/HTTPS の path-based routing を扱える
  - Fargate はインフラ管理負荷を下げつつ container 実行を維持できる
- Cons:
  - App Runner より構成要素は多い
  - IaC とネットワーク設計が必須になる

### Option B: AWS App Runner
- Pros:
  - セットアップが比較的軽い
  - コンテナ化した Web アプリを載せやすい
- Cons:
  - private VPC 接続時に起動レイテンシ上の注意点がある
  - 運用制御や周辺 AWS サービスとの接続で ECS より制約が出やすい

### Option C: Lambda / Amplify Hosting 中心
- Pros:
  - 一部ケースでは構築が容易
  - 静的配信に向く
- Cons:
  - Next.js の self-hosting 前提や SSR 実行モデルとの整合が薄い
  - 本件の認証・DB 接続・長期運用の制御を考えると前提が増える

## Rationale
- Next.js 公式ドキュメントでは self-hosting 時に `next start` で image optimization や middleware を含む動作が可能で、単一 Docker image を複数環境へ昇格させる運用も取れる
- ECS は長時間稼働の stateless service を維持し、失敗 task の置き換えを行う標準基盤である
- ECS + ALB は HTTP/HTTPS の ingress、health check、autoscaling と相性が良い
- CloudWatch Logs と `awslogs` driver により Fargate ログの集約が素直で、運用監視の初期要件を満たしやすい
- Secrets Manager は hard-coded secret を避け、ランタイムでの安全な秘密情報提供に適している

## Consequences
- Impacted Specs:
  - story-0005
  - story-0018
  - story-0019
- Impacted Tests:
  - deployment verification
  - health check / smoke test
  - logging verification
- Operational concerns:
  - Docker build、ALB target group health check、autoscaling policy、CloudWatch alarm 設計が必要
  - ロールバックは task definition revision / service update 単位で管理する必要がある

- 初期導入のため不要
